### Deliberation 2 — OI-015-02 / B9-001: Amendment Path Routing Algorithm

**Question:** Is amendment path selection system-determined, actor-determined, or a hybrid?

**Options presented:**
- Model A — Actor-determined (staff selects all paths)
- Model B — System-determined (classification algorithm assigns path; not derivable from Canon without invention)
- Model C — Hybrid: system-locked for Canon-explicit cases; actor-selects for the residual

**Decision: Model C (Hybrid) confirmed by Architect.**

**Rationale:** The Canon is explicit on two cases — date/room changes post-S4 require re-entry (Path 3); billing model changes require FOM authority (Path 2 minimum). These are system-locked. For everything else, the guest's or agent's intent determines the appropriate path, and intent interpretation is a human judgement that the system cannot reliably classify from amendment data alone. The hybrid avoids Canon invention while preserving system-enforced discipline where the Canon is clear.

**Changes applied to DEV-SPEC-001-Part6.md:**
- B9-001 gap block in §6.6.2 AmendmentService replaced with the Model C hybrid routing table
- `amend()` method description updated: system-locked paths reject conflicting caller-supplied path; actor-selected paths are accepted and authority-enforced without silent rerouting
- Appendix A cleared — B9-001 is resolved; no open clarification requests remain
